# 1️⃣ High-Level Architecture You Should Build

Think in 6 layers:

```
Corpus → Ingestion → Structured Extraction → Validation → Deduplication →
Memory Graph Store → Retrieval API → Visualization UI
```

Your system should produce:

* Entities
* Claims
* Evidence
* Time validity
* Merge history
* Version tracking

This is not an LLM demo.
This is a **memory system with auditability**.

# 2️⃣ Recommended Tech Stack (Best Fit for This Project)

## 🔹 Language

**Python**
Why:

* Best LLM ecosystem
* Best graph tooling
* Best experimentation speed
* Neo4j drivers mature
* Easy backend API

---

## 🔹 Storage

### Graph Store (Primary Choice)

**Neo4j Community Edition (local)**

Why:

* Natural fit for memory graph
* Built-in visualization (Neo4j Browser)
* Cypher query language
* Easy to demonstrate merges + relationships

Alternative:

* Postgres with adjacency tables (harder to show visually)
* NetworkX (too toy-level)

Use Neo4j.

Layer10 cares about deterministic pipelines, not agents.

# 🔹 B) Ontology Design (CRITICAL)

You must define:

## Entities

* Person
* Issue
* Project
* Component
* Organization

## Claims

* AssignedTo
* StatusChange
* Decision
* Mentions
* DependsOn

## Evidence

* source_id
* excerpt
* timestamp
* offset

Graph structure:

```
(Person)-[:MADE_CLAIM]->(Claim)-[:SUPPORTED_BY]->(Evidence)
(Claim)-[:ABOUT]->(Entity)
```

This is the core of Layer10 thinking.

---

# 🔹 C) Structured Extraction

DO NOT just dump raw JSON.

Do this:

1. Prompt model for:

   * Entities
   * Claims
   * Relations
   * Evidence span offsets

2. Validate with Pydantic models

3. If invalid → repair pass

Use:

```python
from pydantic import BaseModel
```

Make extraction deterministic:

* Temperature = 0
* Fixed schema
* Retry if invalid

---

# 🔹 D) Deduplication Strategy

You need 3 layers:

## 1️⃣ Artifact Dedup

Hash content
Normalize whitespace
Remove quoted text blocks

## 2️⃣ Entity Canonicalization

* Normalize emails
* Lowercase names
* Embedding similarity threshold (0.9+)

## 3️⃣ Claim Dedup

Same:

* Subject
* Predicate
* Object
* Time window

Store:

* Support evidence list
* Confidence score

Merges must be reversible.

Create:

```
MergeEvent node:
- merged_at
- reason
- similarity_score
```

That is very Layer10-like thinking.

---

# 🔹 E) Time Handling

Two timestamps:

* Event time (when email written)
* Validity time (when claim true)

For example:
"Issue is assigned to John"
Later:
"Issue reassigned to Sarah"

Old claim:
valid_until = reassignment time

Current claim:
valid_until = null

This is huge.
Most candidates miss this.

---

# 🔹 F) Retrieval API

Implement:

```python
def answer(question: str):
    1. Extract candidate entities via embeddings
    2. Fetch related claims
    3. Rank by:
        - recency
        - confidence
        - evidence count
    4. Return context pack:
        - Claims
        - Evidence snippets
        - Entity graph neighborhood
```

Never return something without evidence.

Every answer must include:

* Evidence excerpt
* Source ID
* Timestamp
